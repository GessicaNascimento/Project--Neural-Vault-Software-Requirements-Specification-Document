# Neural Vault — Enterprise Smart Memory & Knowledge Isolation Engine

## 1. Executive Summary
**Neural Vault** is a conceptual software system designed under the rigorous guidelines of **Ian Sommerville's Software Engineering framework (2007)**. The solution addresses the critical problem of corporate data fragmentation and institutional memory loss across isolated silos (such as chat history, emails, and meeting minutes). By implementing a secure **Retrieval-Augmented Generation (RAG)** architecture, the system centralizes organizational knowledge while strictly mitigating AI hallucinations and ensuring absolute legal compliance.

---

## 2. Dev.Studies Report & Core Contributions
*This repository serves as a comprehensive technical portfolio documenting the requirements engineering, structural domain modeling, and system design processes. All models and specifications were developed for educational and research advancement in complex software architectures.*

### Architectural & Engineering Focus:
* **System Architecture Design:** Responsible for designing the application's core structural layout using a **Multi-Tier Layered Architecture**, successfully isolating external ingestion triggers, asynchronous application processes, and intelligent persistence engines.
* **Security & Privacy Layering:** Spearheaded the architectural design of the system's security perimeter, implementing a structural Named Entity Recognition (NER) NLP pipeline for data anonymization and a strict mathematical constraint mapping a **Cosine Similarity threshold $\ge$ 85%** to shield the LLM from hallucinations.
* **Requirements Engineering & Review:** Led the elicitation, mapping, and formal validation of the system's **Functional Requirements (FR)**, ensuring seamless traceability between enterprise operational dependencies and backend engineering derivations.
